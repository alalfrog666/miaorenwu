# MemPalace 資安風險審查報告

**審查對象**: https://github.com/milla-jovovich/mempalace  
**審查日期**: 2026-04-08  
**專案性質**: 本地 AI 記憶系統，使用 ChromaDB 儲存對話歷史與知識  

---

## 一、整體評估

| 項目 | 評級 | 說明 |
|------|------|------|
| 資料儲存安全性 | 中等風險 | 所有資料儲存於本地，無雲端傳輸，但缺乏存取控制 |
| 輸入驗證 | 中等風險 | 部分參數未經過濾直接傳入資料庫查詢 |
| 程式碼執行風險 | 低風險 | 未發現 `eval()`/`exec()` 或外部指令注入 |
| 依賴套件風險 | 低風險 | 依賴精簡（chromadb、pyyaml），無高風險套件 |
| 網路安全 | 低風險 | 無外部 API 呼叫，無網路傳輸 |
| 認證與授權 | 高風險 | MCP Server 完全無認證機制 |

**綜合風險等級：中等**

---

## 二、已發現的安全風險

### 風險 1：MCP Server 無認證授權機制（高風險）

`mcp_server.py` 提供 19 個工具，但**完全沒有認證或授權機制**。任何透過 stdin 連接的 MCP 呼叫者都可以：

- 查詢所有已儲存的記憶內容
- 新增或刪除任意資料
- 修改知識圖譜
- 以任意代理人名義撰寫日記

**影響**：如果其他未授權的程式取得 MCP 連線存取權，可讀取或竄改所有記憶資料。

**緩解因素**：MCP Server 透過 stdio 通訊，實際上只有啟動它的 AI 工具（如 Claude Code）能存取，攻擊面有限。

---

### 風險 2：輸入參數未經驗證（中等風險）

多個工具函式直接接受使用者輸入而未進行驗證：

```python
# 使用者提供的 wing 參數直接用於 ChromaDB 查詢
if wing:
    kwargs["where"] = {"wing": wing}

# 知識圖譜接受任意字串
def tool_kg_add(subject: str, predicate: str, object: str, ...):
    triple_id = _kg.add_triple(subject, predicate, object, ...)
```

**影響**：雖然 ChromaDB 的查詢語法不像 SQL 容易注入，但缺乏驗證仍可能導致非預期行為。

---

### 風險 3：檔案路徑遍歷風險（中等風險）

`cli.py` 和 `miner.py` 中的路徑處理存在潛在問題：

- 使用者提供的路徑透過 `Path.expanduser()` 展開，但未驗證是否超出預期目錄
- `os.walk()` 預設會跟隨符號連結（symlinks），可能被利用來存取預期外的目錄
- `shutil.copytree()` 和 `shutil.rmtree()` 操作若遇到惡意符號連結可能造成危害

---

### 風險 4：型別強制轉換漏洞（低風險）

MCP Server 中的型別處理邏輯：

```python
for key, value in list(tool_args.items()):
    if declared_type == "integer" and not isinstance(value, int):
        tool_args[key] = int(value)
```

若 schema 定義被繞過或不匹配，可能導致型別混淆。

---

### 風險 5：Hook 腳本安全性（低風險）

`mempal_save_hook.sh` 和 `mempal_precompact_hook.sh`：

- **正面**：Session ID 有進行清理（僅允許英數、底線、連字號）
- **正面**：未發現外部 API 呼叫
- **輕微疑慮**：日誌檔案會持續累積，缺乏日誌輪替機制

---

## 三、正面安全特性

1. **完全本地運作**：所有資料儲存在本地，不傳送至雲端
2. **依賴套件精簡**：僅依賴 `chromadb` 和 `pyyaml`，減少供應鏈攻擊風險
3. **使用 `yaml.safe_load()`**：正確避免 YAML 反序列化攻擊
4. **未使用 `eval()`/`exec()`**：無程式碼注入風險
5. **MIT 授權**：開源透明，可自行審查
6. **Drawer ID 使用雜湊**：防止直接猜測 ID

---

## 四、適用性建議

### 適合使用的情境

- 你想要一個**完全本地**的 AI 記憶系統，不想將對話資料上傳雲端
- 你主要在**個人電腦**上使用，不需要多人共用
- 你使用 Claude Code 或其他支援 MCP 的工具
- 你願意接受一個相對早期的開源專案

### 不建議使用的情境

- 你處理**高度機密**的資料（因缺乏加密與存取控制）
- 你需要在**共享伺服器**上使用（MCP Server 無認證機制）
- 你需要**企業級**的安全保障

### 使用前建議

1. **確認環境安全**：確保只有你自己能存取 MCP Server 的 stdio 通道
2. **定期備份**：`~/.mempalace/` 目錄應定期備份
3. **注意儲存內容**：避免將密碼、API 金鑰等敏感資料存入記憶系統
4. **監控磁碟用量**：ChromaDB 的向量資料庫可能隨時間增長
5. **保持更新**：定期更新至最新版本以獲得安全修正

---

## 五、結論

MemPalace 是一個**設計理念良好**的本地記憶系統。其最大的安全優勢在於**完全本地運作**，不會將你的資料傳送至任何外部伺服器。主要的安全疑慮集中在 MCP Server 缺乏認證機制和輸入驗證不足，但在**個人使用**的場景下，這些風險是可控的。

**建議**：如果你是在個人電腦上使用，且不處理極度敏感的資料，這個工具**適合使用**。
